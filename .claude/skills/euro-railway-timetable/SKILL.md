## Data Location

Merged GTFS CSV files are at `travel-plan-web-next/euro-railway-timetable/` relative to the repo root:

```
euro-railway-timetable/
  agency.txt
  attributions.txt
  calendar_dates.txt
  calendar.txt
  feed_info.txt
  routes.txt
  shapes.txt
  stop_times.txt
  stops.txt
  transfers.txt
  trips.txt
```

Source IDs are prefixed to prevent collisions:

- `eu:` = Eurostar feed rows
- `fr:` = French (SNCF) feed rows
- `de:` = German GTFS feed rows

Examples: `eu:9002-0314`, `fr:OCESN105241...`, `de:810944`

## DuckDB Setup

If DuckDB is not pre-installed, install it first:

```bash
# macOS
brew install duckdb

# Linux
curl -L https://github.com/duckdb/duckdb/releases/latest/download/duckdb_cli-linux-amd64.zip -o duckdb.zip && unzip duckdb.zip
```

Verify: `duckdb --version`

## How to Run Non-Interactive Queries

Always use `duckdb -c`. Never open the interactive shell.

```bash
# Single inline query (run from repo root)
duckdb -c "SELECT ... FROM read_csv('travel-plan-web-next/euro-railway-timetable/trips.txt', header=true) ..."

# Multi-line query
duckdb -c "
  SELECT s.stop_name, st.arrival_time, st.departure_time
  FROM read_csv('travel-plan-web-next/euro-railway-timetable/stop_times.txt', header=true, types={'arrival_time': 'VARCHAR', 'departure_time': 'VARCHAR'}) st
  JOIN ...
"
```

## Important Data Caveats

- **Time fields**: always read `stop_times.txt` with `types={'arrival_time': 'VARCHAR', 'departure_time': 'VARCHAR'}`.
- **Merged `trips.txt` enrichments**:
  - `train_brand` is populated for French and Eurostar rows in merged output
  - French (`fr:`) `trip_headsign` is normalized to `"<BRAND> <NUMBER>"` (for example `TGV 9242`, `TER 860001`)
  - Eurostar (`eu:`) `trip_headsign` is normalized to `"EST <NUMBER>"` (for example `EST 9002`)
  - German (`de:`) rows usually have no train number fields; rely on `trip_id`
- **Best cross-source train identifier**: `trip_id` (always present, prefixed)
- **Date range in merged calendar_dates**: 20260307 to 20260831

For train-number lookups across sources, prefer:

```sql
COALESCE(NULLIF(t.trip_headsign::VARCHAR, ''), NULLIF(t.trip_short_name::VARCHAR, ''))
```

## Full Train Number (Brand + Number)

In merged data, use `trip_headsign` as the first choice because it is already normalized for French and Eurostar.

```sql
SELECT
  split_part(t.trip_id, ':', 1) AS source_prefix,
  t.trip_id,
  t.train_brand,
  COALESCE(NULLIF(t.trip_headsign::VARCHAR, ''), NULLIF(t.trip_short_name::VARCHAR, '')) AS full_train_number
FROM read_csv('travel-plan-web-next/euro-railway-timetable/trips.txt', header=true) t
ORDER BY source_prefix, full_train_number
LIMIT 100;
```

If you need only branded numbers:

```sql
SELECT DISTINCT t.trip_headsign::VARCHAR AS full_train_number
FROM read_csv('travel-plan-web-next/euro-railway-timetable/trips.txt', header=true) t
WHERE NULLIF(t.train_brand, '') IS NOT NULL
ORDER BY full_train_number;
```

## Filter by Operator

You can filter by operator using the prefixed IDs in merged files.

- `fr:` -> French/SNCF
- `eu:` -> Eurostar
- `de:` -> German

Operator filter pattern:

```sql
WHERE split_part(t.trip_id, ':', 1) = 'fr'   -- French
-- or 'eu' / 'de'
```

Example (`french`, `eurostar`, `germany` style input):

```sql
WITH trips_labeled AS (
  SELECT
    t.*,
    CASE split_part(t.trip_id, ':', 1)
      WHEN 'fr' THEN 'french'
      WHEN 'eu' THEN 'eurostar'
      WHEN 'de' THEN 'germany'
      ELSE split_part(t.trip_id, ':', 1)
    END AS operator
  FROM read_csv('travel-plan-web-next/euro-railway-timetable/trips.txt', header=true) t
)
SELECT operator, trip_id, trip_headsign, train_brand
FROM trips_labeled
WHERE operator = 'french'
ORDER BY trip_id
LIMIT 50;
```

## Run Merge Script

Merged data is generated by `travel-plan-web-next/scripts/merge_gtfs.py`.

Run from `travel-plan-web-next/`:

```bash
# Default sources: french + eurostar + german
python3 scripts/merge_gtfs.py --base-dir . --output euro-railway-timetable

# Auto-discover all *-railway-timetable folders
python3 scripts/merge_gtfs.py --base-dir . --output euro-railway-timetable --default-load

# Recommended: run verification checks after merge
python3 scripts/merge_gtfs.py --base-dir . --output euro-railway-timetable --default-load --verify
```

Optional explicit source list:

```bash
python3 scripts/merge_gtfs.py --base-dir . --output euro-railway-timetable --sources french-railway-timetable:fr eurostar-railway-timetable:eu german-railway-timetable:de
```

## Schema Reference (Core Files)

### `trips.txt`
| Column | Description |
|--------|-------------|
| route_id | References routes.txt |
| service_id | References calendar_dates.txt |
| trip_id | Stable trip identifier (prefixed by source) |
| trip_headsign | Destination or train number depending on source |
| train_brand | Brand label in merged output (`TGV`, `TER`, `EST`, etc.; empty for some feeds) |
| trip_short_name | Train number for some feeds (not universal) |
| direction_id | Direction flag if provided |

### `stop_times.txt`
| Column | Description |
|--------|-------------|
| trip_id | References trips.txt |
| arrival_time / departure_time | HH:MM:SS strings (can exceed 24:00:00) |
| stop_id | References stops.txt |
| stop_sequence | Stop order within trip |

### `stops.txt`
| Column | Description |
|--------|-------------|
| stop_id | Station/stop identifier (prefixed by source) |
| stop_name | Human-readable station name |
| stop_lat / stop_lon | Coordinates |
| stop_timezone | Timezone where provided |

### `calendar_dates.txt`
| Column | Description |
|--------|-------------|
| service_id | References trips.txt |
| date | YYYYMMDD integer |
| exception_type | 1=service runs, 2=service removed |

## Common Query Patterns

### Find trips between two stations on a date (cross-Europe)
```sql
SELECT
  split_part(dep.trip_id, ':', 1) AS source_prefix,
  dep.trip_id,
  COALESCE(NULLIF(t.trip_headsign::VARCHAR, ''), NULLIF(t.trip_short_name::VARCHAR, '')) AS train_ref,
  s_dep.stop_name AS from_stop,
  dep.departure_time,
  s_arr.stop_name AS to_stop,
  arr.arrival_time
FROM read_csv('travel-plan-web-next/euro-railway-timetable/stop_times.txt', header=true, types={'arrival_time': 'VARCHAR', 'departure_time': 'VARCHAR'}) dep
JOIN read_csv('travel-plan-web-next/euro-railway-timetable/stop_times.txt', header=true, types={'arrival_time': 'VARCHAR', 'departure_time': 'VARCHAR'}) arr ON dep.trip_id = arr.trip_id
JOIN read_csv('travel-plan-web-next/euro-railway-timetable/stops.txt', header=true) s_dep ON dep.stop_id = s_dep.stop_id
JOIN read_csv('travel-plan-web-next/euro-railway-timetable/stops.txt', header=true) s_arr ON arr.stop_id = s_arr.stop_id
JOIN read_csv('travel-plan-web-next/euro-railway-timetable/trips.txt', header=true) t ON dep.trip_id = t.trip_id
JOIN read_csv('travel-plan-web-next/euro-railway-timetable/calendar_dates.txt', header=true) cd ON t.service_id = cd.service_id
WHERE s_dep.stop_name ILIKE '%London%'
  AND s_arr.stop_name ILIKE '%Paris%'
  AND dep.stop_sequence < arr.stop_sequence
  AND cd.date = 20260314
  AND cd.exception_type = 1
ORDER BY dep.departure_time
LIMIT 50;
```

### Full timetable for a trip_id on a specific date
```sql
SELECT
  st.stop_sequence,
  s.stop_name,
  st.arrival_time,
  st.departure_time
FROM read_csv('travel-plan-web-next/euro-railway-timetable/stop_times.txt', header=true, types={'arrival_time': 'VARCHAR', 'departure_time': 'VARCHAR'}) st
JOIN read_csv('travel-plan-web-next/euro-railway-timetable/stops.txt', header=true) s ON st.stop_id = s.stop_id
JOIN read_csv('travel-plan-web-next/euro-railway-timetable/trips.txt', header=true) t ON st.trip_id = t.trip_id
JOIN read_csv('travel-plan-web-next/euro-railway-timetable/calendar_dates.txt', header=true) cd ON t.service_id = cd.service_id
WHERE st.trip_id = 'eu:9002-0314'
  AND cd.date = 20260314
  AND cd.exception_type = 1
ORDER BY st.stop_sequence;
```

### Search by train number-like value (where available)
```sql
SELECT
  split_part(t.trip_id, ':', 1) AS source_prefix,
  t.trip_id,
  t.service_id,
  COALESCE(NULLIF(t.trip_headsign::VARCHAR, ''), NULLIF(t.trip_short_name::VARCHAR, '')) AS train_ref
FROM read_csv('travel-plan-web-next/euro-railway-timetable/trips.txt', header=true) t
WHERE COALESCE(NULLIF(t.trip_headsign::VARCHAR, ''), NULLIF(t.trip_short_name::VARCHAR, '')) = '9002'
ORDER BY t.trip_id;
```

### List all service dates for one trip_id
```sql
SELECT cd.date
FROM read_csv('travel-plan-web-next/euro-railway-timetable/trips.txt', header=true) t
JOIN read_csv('travel-plan-web-next/euro-railway-timetable/calendar_dates.txt', header=true) cd ON t.service_id = cd.service_id
WHERE t.trip_id = 'eu:9002-0314'
  AND cd.exception_type = 1
ORDER BY cd.date;
```

### Top matching stations for a keyword
```sql
SELECT DISTINCT stop_id, stop_name, stop_timezone
FROM read_csv('travel-plan-web-next/euro-railway-timetable/stops.txt', header=true)
WHERE stop_name ILIKE '%Brussels%'
ORDER BY stop_name
LIMIT 50;
```

## Query Tips

- Use repo-root-relative paths: `travel-plan-web-next/euro-railway-timetable/*.txt`.
- Start with station name matching (`ILIKE`) and then join to trips.
- Prefer `trip_id` in answers when the source has no train number field.
- Include `split_part(trip_id, ':', 1)` in outputs to show source (`eu`, `fr`, `de`).
- Always filter `calendar_dates.exception_type = 1` for active services.
- If the user gives only city names, use fuzzy station lookup first, then reuse exact stop names in route queries.
